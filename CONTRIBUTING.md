# Contributing to Arcana 🔮

First off, thank you for considering contributing to Arcana! It's people like you who make this tool better for everyone seeking **Digital Sovereignty**.

To keep the project organized and maintain high code quality, please follow these guidelines.

## Our Philosophy 🛡️
- **Privacy First:** Never implement features that compromise user data.
- **Minimalism:** We prefer simple, efficient solutions over heavy libraries.
- **Performance:** If it's slow, it's a bug.

## How Can I Contribute?

### Reporting Bugs 🐛
- Use the **Bug Report** issue template.
- Describe the problem clearly and provide steps to reproduce it.
- Include environment details (Browser, OS, or if you're using the Capacitor mobile version).

### Suggesting Enhancements 💡
- Check if the feature hasn't been suggested already.
- Open an issue with the label `enhancement` to discuss the idea before starting to code.

### Pull Requests (PRs) 🚀
1. Fork the repo and create your branch from `main`.
2. Ensure your code follows the project's style (Tailwind classes, React Hooks, etc.).
3. Update documentation if you're changing functionality.
4. **Keep your commit messages clear and concise.** Just describe what you did (e.g., "Fix deck header alignment" or "Add search to library") so anyone can understand the history.

### Code of Conduct
By participating in this project, you agree to maintain a respectful and collaborative environment.
